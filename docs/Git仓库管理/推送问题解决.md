# Git 推送问题解决指南

## 当前状态

✅ **本地提交已完成**
- 所有文档整理更改已提交到本地仓库
- 数据安全，不会丢失

❌ **远程推送失败**
- 无法连接到 GitHub（网络问题）

## 解决方案

### 方案1: 配置 Git 使用系统代理（推荐）

如果你的 VPN 有 HTTP/HTTPS 代理端口，可以配置 Git 使用代理：

```bash
# 查看你的 VPN 代理端口（通常在 VPN 设置中）
# 假设代理是 127.0.0.1:7890（常见端口）

# 设置 HTTP 代理
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 然后重试推送
git push origin main

# 如果不需要代理了，可以取消设置
# git config --global --unset http.proxy
# git config --global --unset https.proxy
```

### 方案2: 使用 SSH 连接（如果已配置 SSH 密钥）

```bash
# 将远程仓库地址改为 SSH
git remote set-url origin git@github.com:xlkm/IndexTTS2.git

# 然后推送
git push origin main
```

### 方案3: 稍后重试

如果网络暂时不稳定，可以稍后再试：

```bash
git push origin main
```

### 方案4: 检查 VPN 设置

确保 VPN 对命令行工具生效：
- 某些 VPN 只对浏览器生效
- 需要配置系统代理或 VPN 的全局模式

## 验证本地提交

```bash

git log --oneline -5

# 查看当前状态
git status

# 查看所有分支
git branch -a
```
