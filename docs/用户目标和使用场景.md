# 用户目标和使用场景

## 📋 基本信息

**用户身份**: 家政保洁服务提供者  
**业务模式**: 通过抖音短视频获取客户，客户线下联系进行保洁服务

## 🎯 核心目标

### 主要需求
使用 IndexTTS2 为自己的视频快速生成配音，替代原来需要1-2小时的口播录制过程。

### 使用场景
1. **文案准备**: 编辑好视频文案
2. **快速配音**: 使用 IndexTTS2 生成配音音频
   - 支持情绪/语调微调
   - 保持个人声音特色（语音克隆）
3. **视频制作**: 将生成的配音导入剪映进行视频编辑
4. **发布推广**: 在抖音发布短视频获取客户

### 工作流程（当前）
```
文案编辑 → 自己录制口播（1-2小时） → 剪映编辑 → 发布抖音
```

### 目标工作流程
```
文案编辑 → IndexTTS2生成配音（几分钟） → 剪映编辑 → 发布抖音
```

## 🚀 功能需求

### 核心功能
1. **文本转语音**: 将文案转换为自然语音
2. **语音克隆**: 使用自己的声音样本，生成具有个人特色的配音
3. **情绪控制**: 可以调整语音的情绪、语调、语速等参数
4. **快速生成**: 相比人工录制，大幅缩短时间

### 扩展功能（后期）
1. **工作流自动化**: 
   - 批量处理多个文案
   - 自动生成配音
   - 自动导入剪映或导出标准格式
2. **模板管理**: 
   - 保存常用的情绪/语调配置
   - 快速应用模板
3. **批量处理**: 
   - 一次处理多个视频的配音需求

## 💡 使用特点

### 优势
- ⏱️ **时间节省**: 从1-2小时缩短到几分钟
- 🎭 **情绪可控**: 可以调整情绪，适应不同内容
- 🔄 **可重复**: 可以反复调整，直到满意
- 📝 **文案驱动**: 修改文案即可重新生成，无需重新录制

### 注意事项
- 需要准备自己的声音样本用于语音克隆
- 需要学习基本的IndexTTS2使用方法
- 生成的音频需要符合剪映的格式要求

## 🎬 剪映集成需求

### 音频格式要求
- **格式**: WAV 或 MP3
- **采样率**: 建议 44.1kHz 或 48kHz（剪映常用）
- **声道**: 单声道或立体声
- **时长**: 根据文案长度自动生成

### 导出选项
- 直接导出剪映兼容格式
- 保留原始高质量版本
- 支持批量导出

## 📊 业务价值

### 时间成本
- **原来**: 1-2小时/视频
- **现在**: 5-10分钟/视频
- **节省**: 约90%的时间

### 内容产出
- 可以更快速地制作多个视频
- 提高内容更新频率
- 增加抖音曝光机会

### 质量提升
- 可以反复调整，确保最佳效果
- 情绪控制更精准
- 文案修改后快速重新生成

## 🔧 技术适配

### 环境特点
- **设备**: Mac M2（Apple Silicon）
- **系统**: macOS
- **GPU**: Apple M2（使用MPS而非CUDA）

### 注意事项
- IndexTTS2需要适配MPS（Metal Performance Shaders）
- 可能需要使用CPU推理（速度较慢但可用）
- 考虑使用云端GPU服务（如需要更快速度）
